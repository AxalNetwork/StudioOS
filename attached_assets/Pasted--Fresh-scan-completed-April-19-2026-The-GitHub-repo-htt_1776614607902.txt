### Proposed Profile Structure (Optimized for Value Creation)

**Core Principle**: Every user profile should be a **value-adding asset** to the network — feeding PMF signals, deal flow, partner matching, voting, Studio Ops, and Market Intelligence.

#### 1. Common Core Profile (all users)
- `users` table fields (extend existing):
  - id, email, full_name, role, avatar_url (R2), bio, location, linkedin_url, twitter_url, website
  - kyc_status, verified_badges, network_score, total_earnings, referrer_id
  - last_active, joined_at

#### 2. Role-Specific Profile Extensions (separate tables or JSON extensions)

**Founder Profile** (`founder_profiles` table)
- Startup focus areas, previous exits, current idea/pitch deck link (R2)
- PMF signals they can contribute (problem validation, customer interviews, waitlist data)
- Skills: product, tech stack, go-to-market experience

**Partner / Investor / LP Profile** (`investor_profiles` table)
- Investment thesis, check size, stage preference, sector focus
- Portfolio companies (links), past exits, co-investment interest
- Deal flow contribution score, voting history, syndication activity

**Operator / Advisor Profile** (`operator_profiles` table)
- Expertise areas, past roles, availability (hours/week)
- Skills matrix (for matching to spin-outs)
- Past contributions (Studio Ops tasks completed, mentoring hours)

**Service Provider Profile** (`service_profiles` table)
- Services offered (legal, accounting, tech, marketing, etc.)
- Pricing tiers, past clients, case studies (R2)
- Availability and capacity

This structure allows:
- **PMF data** → Founders contribute validation signals
- **Deal flow** → Investors + Operators flag opportunities
- **Partner matching** → AI uses skills/thesis/expertise for recommendations
- **Voting & Intelligence** → Weighted input based on role + track record
