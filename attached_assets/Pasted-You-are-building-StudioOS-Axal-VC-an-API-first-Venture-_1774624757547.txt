You are building **StudioOS (Axal VC)** — an API-first Venture Studio Operating System that enables launching startups in 30 days.

Your task is to generate a **full-stack application** with backend APIs, database schema, and basic frontend dashboards.

---

# 🧠 PRODUCT OVERVIEW

StudioOS is a modular platform with 4 layers:

1. Data Layer (market intelligence, portfolio tracking)
2. Decision Layer (AI scoring, diligence)
3. Execution Layer (legal, incorporation, workflows)
4. Network Layer (partners, deal flow, referrals)

Core goal:
→ Turn ideas into funded startups in 30 days via automation.

---

# ⚙️ CORE FEATURES TO BUILD (MVP PRIORITY)

## 1. Startup Scoring Engine (Gatekeeper)

* Endpoint: POST /scoreStartup

* Input:

  * market (tam, urgency, trend)
  * team (experience, execution, network)
  * product (mvpTimeDays, complexity, dependencies)
  * financials (costToMVP, timeToRevenueMonths, burnRisk)
  * axalFit
  * distribution (channels, virality)

* Output:

  * score (0–100)
  * tier (TIER_1, TIER_2, REJECT)
  * breakdown

* Logic:

  * Market: 25%
  * Team: 20%
  * Product: 15%
  * Capital: 15%
  * Fit: 15%
  * Distribution: 10%

---

## 2. Deal Flow CRM

* Entities:

  * Startup
  * Founder
  * Partner
  * Deal
* Features:

  * Track status: applied → scored → active → funded
  * Store scoring results
  * Link partners to startups

---

## 3. Partner Matchmaking Engine

* Endpoint: POST /matchPartners
* Input:

  * startup needs (sector, capital, expertise)
  * partner profile
* Output:

  * ranked list of partners

---

## 4. Legal & Incorporation Engine (Mock MVP)

* Endpoint: POST /incorporate
* Simulate:

  * company creation
  * document generation
* Store:

  * entity
  * documents

---

## 5. Deal Memo Generator

* Endpoint: POST /generateMemo
* Input: startup data
* Output:

  * structured memo:

    * problem
    * solution

    * decision

---

## 6. Capital Call System (Basic)

* Endpoint: POST /capitalCall
* Input:

  * startup_id

* Output:

  * list of participating partners
