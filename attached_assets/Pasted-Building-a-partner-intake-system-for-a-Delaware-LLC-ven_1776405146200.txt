### 1. The D1 SQLite Schema
In your Replit project, create a `schema.sql` file. This structure handles the transition from an anonymous visitor to a "Verified Partner."

```sql
-- Track user sessions and verification
CREATE TABLE users (
    id TEXT PRIMARY KEY, 
    email TEXT UNIQUE, 
    otp TEXT, 
    is_verified INTEGER DEFAULT 0,
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP
);

-- Store the specific StudioOS partnership data
CREATE TABLE partner_profiles (
    user_id TEXT PRIMARY KEY,
    persona TEXT, -- Investor (LP/Syndicate), Legal, Advisory (Operator), Liquidity
    legal_entity_name TEXT,
    entity_type TEXT, -- e.g., Delaware C-Corp, LLC, Individual
    tax_id TEXT, -- EIN
    signatory_name TEXT,
    signatory_title TEXT,
    referral_code TEXT,
    admin_approval_status TEXT DEFAULT 'pending', -- pending, verified, rejected
    FOREIGN KEY(user_id) REFERENCES users(id)
);

-- Store chat history for stateful AI memory
CREATE TABLE chat_history (
    session_id TEXT,
    role TEXT, -- 'user' or 'assistant'
    content TEXT,
    timestamp DATETIME DEFAULT CURRENT_TIMESTAMP
);
```

### 2. The Cloudflare AI Worker Logic
The "Brain" of your system needs to change its **System Prompt** based on the user's state. You can prompt Replit to generate a script with this logic:

* **STATE 0 (Anonymous):** The AI acts as a "Gatekeeper." It explains the StudioOS model and tries to categorize the user into the five categories (Investor, Legal, Advisory, Liquidity, or Project Founder).
* **STATE 1 (Verifying):** Once the persona is found, the UI triggers the email OTP. The Worker saves the email to D1.
* **STATE 2 (Profiling):** After verification, the AI becomes a "Legal Intake Officer." It specifically asks for the **Delaware LLC/C-Corp** details (EIN, Signatory) required for Axal's automated legal engine.

**To approve a partner:**
```sql

SET admin_approval_status = 'verified'
```
