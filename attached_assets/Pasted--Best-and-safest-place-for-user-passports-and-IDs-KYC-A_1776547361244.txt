**Best and safest place for user passports and IDs (KYC/AML documents):**

**Cloudflare R2 (private bucket) + metadata only in D1.**

### Why R2 is the best Cloudflare solution
- **Private by default** — you control access with signed URLs (short-lived, e.g. 5–15 minutes).
- **Encrypted at rest** — automatic server-side encryption.
- **Global, low-latency, cheap** — perfect for documents.
- **Compliant-friendly** — you can enable Cloudflare Logs for full audit trail of every access.
- **No public URLs** — unlike Pages or public buckets.
- Better than:
  - D1 (too small, not for binary files)
  - KV (not designed for files)
  - Durable Objects (not for storage)

**Best practice architecture:**
- Store the actual file (passport PDF/JPG) in a **private R2 bucket**.
- Store **only metadata** in D1 (`file_key`, `hash`, `uploaded_at`, `verification_status`, `expires_at`).
- Generate temporary signed URLs only when needed (admin review, Persona/Sumsub verification, audit).
